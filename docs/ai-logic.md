# UNO 在线联机 - AI 玩家逻辑

## 运行位置

AI 逻辑完全在服务端执行，不涉及客户端页面。AI 是一段策略函数，当轮到 AI 时，服务端直接调用并更新 Firestore，与处理真人请求共用同一套数据操作。

## 触发时机

每次服务端更新 `currentPlayerIndex` 后，检查下一个玩家是否是 AI：

- 是真人 → 等待客户端 API 请求
- 是 AI → 服务端立即调用 AI 策略函数执行动作，然后再检查下一个玩家，循环直到轮到真人

需要注意连续多个 AI 的情况（如 3 个 AI 连续出牌），服务端在一次请求中循环处理直到轮到真人为止。

## 策略函数

```
function aiDecide(gameState, hand) → Action
```

输入：当前游戏公开状态 + AI 自己的手牌
输出：一个动作（出牌/摸牌/跳过/接受/质疑/选颜色）

## 第一版：随机策略

### 常规出牌回合

1. 遍历手牌，找出所有合法牌（匹配颜色、数字/符号，或万能牌）
2. 有合法牌 → 随机选一张打出
3. 如果打出的是万能牌 → 随机选一个颜色
4. 没有合法牌 → 摸牌
5. 摸到的牌如果能出 → 随机决定出或不出
6. 不出 → 跳过

### 被 +2 时

1. 手中有 +2 → 打出叠加
2. 没有 +2 → 接受摸牌

### 被 +4 时

1. 随机决定接受或质疑（50% 概率）

### 选庄摸牌

1. 调用摸牌，无需决策

### 庄家发牌

1. 如果 AI 是庄家，自动触发发牌

## 后期可升级方向

| 难度 | 策略 |
|------|------|
| 简单 | 随机出合法牌（第一版） |
| 中等 | 优先出功能牌 > 出手牌中最多颜色的牌 > 保留万能牌 |
| 困难 | 加入记牌、根据对手手牌数量调整策略、合理使用质疑 |

## AI 操作延迟

为了让游戏节奏自然，AI 操作不应瞬间完成。建议每个 AI 动作加 1-2 秒随机延迟，避免多个 AI 连续出牌时界面变化太快。
