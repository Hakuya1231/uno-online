# 房间模块 E2E 用例

对齐文档：[pages.md](../pages.md)

目标：覆盖“能进房间、能加入、能开始、能同步跳转”的主路径，先确保可玩链路稳定，再逐步补细粒度权限/边界用例。

---

## 用例 1：进入首页自动匿名登录 + 创建房间跳转成功

- **Given**
  - 清空浏览器上下文存储（localStorage/IndexedDB），模拟第一次访问
- **When**
  - 打开首页 `/`
  - 等待页面显示“已登录”（或至少 userId 出现）
  - 点击“创建房间”
- **Then**
  - URL 变为 `/room/[roomId]`
  - 房间页显示“玩家列表”
  - 列表包含 1 个玩家，且该玩家显示“(房主)”
  - 不出现 “房间订阅错误：Missing or insufficient permissions.”

---

## 用例 2：分享链接直达房间页，设置昵称并加入（非房主路径）

- **Given**
  - 已有一个房间 `roomId`（可由用例 1 创建得到）
  - 使用第二个浏览器上下文（模拟另一位玩家）
- **When**
  - 直接打开 `/room/[roomId]`（不经过首页）
  - 在昵称组件输入昵称（或点“随机”）
  - 点击“加入房间”
- **Then**
  - 页面提示“加入成功”或出现“你已加入房间，等待房主开始游戏…”
  - 玩家列表人数变为 2，包含第二个玩家昵称
  - 加入后昵称组件不再显示

---

## 用例 3：房主开始游戏后，所有已加入玩家自动跳转到游戏页

- **Given**
  - 房主上下文停留在 `/room/[roomId]`
  - 第二个玩家已加入且也停留在 `/room/[roomId]`
- **When**
  - 房主点击“开始游戏”
- **Then**
  - 房主页面自动跳转到 `/game/[roomId]`
  - 第二个玩家页面也自动跳转到 `/game/[roomId]`
  - 游戏页能显示 `roomId`（作为跳转完成的断言点）

---

## 用例 4：非房主不可见“开始游戏”

- **Given**
  - 第二个玩家在 `/room/[roomId]`
- **When**
  - 观察页面
- **Then**
  - 页面不出现“开始游戏”按钮（仅房主可见）

---

## 用例 5：房主开始游戏按钮启用条件

- **Given**
  - 房主在 `/room/[roomId]`
- **When / Then**
  - 人数 < 2：按钮 disabled
  - `room.status !== "waiting"`：按钮 disabled
  - 人数 ≥ 2 且 `room.status === "waiting"`：按钮 enabled

---

## 用例 6：加入房间幂等（重复加入不应产生重复玩家）

- **Given**
  - 第二个玩家已经加入
- **When**
  - 刷新页面（模拟断网/重进）
  - 再次尝试加入（如果 UI 已隐藏“加入房间”，则改为通过“重新打开房间页 + 观察玩家列表”验证）
- **Then**
  - 不报错（或仍保持“已加入/等待”状态）
  - 玩家列表不会出现重复玩家条目
